  <action>
    Create individual retriever components and the RRF fusion algorithm.

    **retrieval/dense.py — DenseRetriever class:**
    - Wraps `ChromaStore.search()`.
    - Constructor takes `chroma: ChromaStore`, `top_k: int` (from settings.dense_top_k, default 20).
    - Method `retrieve(self, query: str) -> list[ScoredChunk]`: Calls chroma.search(query, k=self.top_k). Returns ScoredChunks with source="dense".
    - Simple wrapper — ChromaStore already handles embedding the query and searching.

    **retrieval/sparse.py — SparseRetriever class:**
    - Wraps `BM25Store.search()`.
    - Constructor takes `bm25: BM25Store`, `top_k: int` (from settings.sparse_top_k, default 20).
    - Method `retrieve(self, query: str) -> list[ScoredChunk]`: Calls bm25.search(query, k=self.top_k). Returns ScoredChunks with source="sparse".

    **retrieval/fusion.py — RRFFusion class:**
    - Implements Reciprocal Rank Fusion algorithm (Cormack et al., 2009).
    - Constructor takes `k: int = 60` (the RRF constant, from settings.rrf_k).
    - Method `fuse(self, result_lists: list[list[ScoredChunk]]) -> list[ScoredChunk]`:
      1. Build a dict mapping chunk_id → cumulative RRF score.
      2. For each result list, iterate chunks with their rank (0-indexed):
         `rrf_score += 1 / (k + rank + 1)` (rank is 1-indexed in the formula, so add 1).
      3. Also maintain chunk_id → Chunk mapping (keep the first occurrence).
      4. Deduplicate: each chunk_id appears once with its total RRF score.
      5. Sort by RRF score descending.
      6. Return as list[ScoredChunk] with source="fused".
    - This is a pure algorithm — no external dependencies, highly testable.

    **tests/unit/test_fusion.py:**
    - Test RRF with known inputs and manually computed expected outputs.
    - Test case 1: Two lists with overlapping chunks — verify fused scores are sum of individual RRF scores, verify deduplication.
    - Test case 2: Two lists with NO overlap — verify all chunks appear, scores are from single list only.
    - Test case 3: Empty list input — returns empty.
    - Test case 4: Single list — RRF of one list equals re-scored version of that list.
    - Test case 5: Verify ordering — highest RRF score first.
    - Create ScoredChunk fixtures with mock Chunk objects for testing.

    Create the HybridRetriever that orchestrates dense + sparse + fusion, and write integration tests proving the full ingest → retrieve pipeline works.

    **retrieval/hybrid.py — HybridRetriever class:**
    - Constructor takes: `dense: DenseRetriever`, `sparse: SparseRetriever`, `fusion: RRFFusion`, `top_k: int` (from settings.final_top_k, default 5).
    - Method `retrieve(self, query: str) -> list[ScoredChunk]`:
      1. Get dense results: `dense_results = self.dense.retrieve(query)`
      2. Get sparse results: `sparse_results = self.sparse.retrieve(query)`
      3. Fuse: `fused = self.fusion.fuse([dense_results, sparse_results])`
      4. Truncate to top_k: `return fused[:self.top_k]`
    - NOTE: No cross-encoder reranking in Phase 1 — deferred to v2 per roadmap decision. The pipeline is: dense + sparse → RRF → top-k. The HybridRetriever architecture supports adding a reranker step later without structural changes.
    - Factory function `create_retriever(chroma: ChromaStore, bm25: BM25Store, settings: Settings | None = None) -> HybridRetriever`: Constructs all sub-components and returns ready-to-use retriever.

    **retrieval/__init__.py:**
