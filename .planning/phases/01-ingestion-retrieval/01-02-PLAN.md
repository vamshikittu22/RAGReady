Purpose: Without indexed data in both dense and sparse stores, retrieval cannot function. The ingestion pipeline is the write path; this plan ensures data flows correctly from raw documents into searchable indexes.

Output: Working ingestion pipeline that takes a file path and produces indexed chunks in both ChromaDB and BM25, with a document manifest for tracking and sync verification.
</objective>

  <action>
    Create the chunking layer that splits extracted text into semantic chunks with rich metadata.

    **ingestion/chunker.py — DocumentChunker class:**
    - Uses LangChain's `RecursiveCharacterTextSplitter` from `langchain_text_splitters` (part of langchain-core).
    - Constructor takes `chunk_size: int = 512`, `chunk_overlap: int = 50` from Settings.
    - Separators: `["\n\n", "\n", ". ", " ", ""]` — paragraph boundaries first (Pitfall 5 prevention).
    - `add_start_index=True` to track position in source document.
    - Method `chunk(self, text: str, document_id: str, source_file: str, page_count: int | None = None) -> list[Chunk]`:
      1. Split text using RecursiveCharacterTextSplitter.
      2. For each split, create a `Chunk` with:
         - `chunk_id`: Use `Chunk.generate_id(document_id, chunk_index, split_text)`
         - `document_id`: passed in
         - `text`: the split text
         - `metadata`: ChunkMetadata with source_document=source_file, page_number=inferred from start_index if page_count available (estimate: start_index / (total_length / page_count)), position_in_doc=start_index, chunk_index=index in list, content_hash=sha256(text)[:16], ingested_at=now
      3. Return list of Chunks.
    - Note: Use `tiktoken` for accurate token counting is optional — RecursiveCharacterTextSplitter uses character count by default. Set `chunk_size=512` as character count (roughly maps to tokens for English text). If you want token-accurate splitting, pass `length_function` using tiktoken, but character-based is fine for v1.

    **ingestion/metadata.py:**
    - Function `compute_content_hash(text: str) -> str`: SHA-256 hash of text, truncated to 16 hex chars.
    - Function `compute_document_id(filename: str, content_hash: str) -> str`: Deterministic document ID from filename + content_hash. Use SHA-256 of f"{filename}:{content_hash}" truncated to 16 hex chars.
    - Function `detect_file_type(file_path: Path) -> str`: Map extension to type string ("pdf", "markdown", "html", "plaintext"). Raise ExtractionError if unsupported.

    **tests/unit/test_chunker.py:**
    - Test chunking a long text (2000+ chars) produces multiple chunks.
    - Test chunk_size and overlap are respected (chunks should be <= chunk_size chars).
    - Test metadata is populated: chunk_id is deterministic (same input = same ID), position_in_doc increases, content_hash is correct.
    - Test short text (< chunk_size) produces exactly 1 chunk.
    - Test empty text raises ChunkingError or returns empty list (choose: return empty list with a warning log).

  <action>
    Create storage adapters for both indexes and the pipeline that orchestrates the full ingestion flow.

    **storage/chroma.py — ChromaStore class:**
    - Uses `chromadb.PersistentClient(path=settings.chroma_persist_dir)` — NOT ephemeral Client (Pitfall 7).
    - Uses `SentenceTransformerEmbeddingFunction(model_name=settings.embedding_model)` from `chromadb.utils.embedding_functions` for automatic embedding at add time.
    - Constructor creates/gets collection named "ragready" with the embedding function.
    - Store embedding model name in collection metadata: `collection.metadata["embedding_model"] = settings.embedding_model` for model mismatch detection (Pitfall 8).
    - Method `add_chunks(self, chunks: list[Chunk]) -> None`: Add chunks to collection. Use chunk.chunk_id as ID, chunk.text as document, chunk.metadata as dict (convert ChunkMetadata to dict, handle datetime serialization to ISO string).
    - Method `delete_by_document(self, document_id: str) -> int`: Delete all chunks with matching document_id from metadata. Use `collection.get(where={"document_id": document_id})` to find IDs, then `collection.delete(ids=found_ids)`. Return count deleted.
    - Method `search(self, query: str, k: int = 20) -> list[ScoredChunk]`: Query collection, return ScoredChunks with source="dense". ChromaDB returns distances — convert to similarity scores (1 - distance for cosine).
    - Method `count(self) -> int`: Return total documents in collection.
    - Method `get_document_ids(self) -> set[str]`: Return unique document_ids from all chunks.

    **storage/bm25_store.py — BM25Store class:**
    - Uses `rank_bm25.BM25Okapi` for scoring.
    - Maintains internal state: `_chunks: list[Chunk]` (ordered), `_tokenized_corpus: list[list[str]]`, `_bm25: BM25Okapi | None`.
    - Tokenization: simple `text.lower().split()` — BM25 doesn't need sophisticated tokenization for this use case.
    - Method `add_chunks(self, chunks: list[Chunk]) -> None`: Append chunks, tokenize, rebuild BM25 index. Call `_save()` to persist.
    - Method `delete_by_document(self, document_id: str) -> int`: Remove chunks matching document_id, rebuild index, persist. Return count deleted.
    - Method `search(self, query: str, k: int = 20) -> list[ScoredChunk]`: Tokenize query, get BM25 scores, return top-k as ScoredChunks with source="sparse". Handle empty index gracefully (return []).
    - Method `count(self) -> int`: Return total chunks.
    - Method `get_document_ids(self) -> set[str]`: Return unique document_ids.
    - **Persistence:** `_save(self)` pickles `_chunks` list to `settings.bm25_persist_path`. `_load(self)` loads from pickle at construction. Handle missing file (fresh start) and corrupted file (log warning, start fresh). This addresses the BM25 persistence concern from STATE.md.
    - **IMPORTANT:** Rebuild `_bm25` index from `_tokenized_corpus` after every add/delete/load. BM25Okapi is constructed once from the full corpus — it cannot be incrementally updated.

    **storage/document_store.py — DocumentStore class:**
    - Tracks ingested documents as a manifest (JSON file at `data/indexes/documents.json`).
    - Method `add_document(self, doc: Document) -> None`: Add to manifest, save to disk.
    - Method `get_document(self, document_id: str) -> Document | None`: Look up by ID.
    - Method `list_documents(self) -> list[Document]`: Return all documents.
    - Method `delete_document(self, document_id: str) -> Document | None`: Remove from manifest, save. Return the deleted Document or None.
    - Method `verify_sync(self, chroma: ChromaStore, bm25: BM25Store) -> dict`: Compare document_ids across all 3 stores. Return `{"in_sync": bool, "manifest_count": int, "chroma_count": int, "bm25_count": int, "missing_from_chroma": set, "missing_from_bm25": set}`. Log warnings if out of sync. (Pitfall 2 prevention).
    - Persistence: Simple JSON file with list of Document dicts. Load on construction, save after every mutation.

    **ingestion/pipeline.py — IngestionPipeline class:**
    - Constructor takes: `extractor_registry: ExtractorRegistry`, `chunker: DocumentChunker`, `chroma: ChromaStore`, `bm25: BM25Store`, `doc_store: DocumentStore`.
    - Method `ingest(self, file_path: Path) -> Document`:
      1. Validate file exists.
      2. Extract text via extractor_registry.
      3. Compute document_id via `compute_document_id(filename, content_hash)`.
      4. Check if already ingested (doc_store.get_document) — if so, log and return existing.
      5. Chunk text via chunker.
      6. Index to ChromaDB (chroma.add_chunks).
      7. Index to BM25 (bm25.add_chunks).
      8. Create Document record, add to doc_store.
      9. Log: "{filename}: {chunk_count} chunks indexed".
      10. Return Document.
    - If step 6 succeeds but step 7 fails, attempt cleanup: delete from chroma. Log error. Raise IndexingError. This ensures atomic dual-indexing (Pitfall 2).
    - Method `delete(self, document_id: str) -> bool`: Delete from chroma, bm25, and doc_store. Return True if found.
    - Factory function `create_pipeline(settings: Settings | None = None) -> IngestionPipeline`: Constructs all dependencies from Settings and returns ready-to-use pipeline.

    **tests/integration/test_ingestion_pipeline.py:**
    - Use tmp_path for all storage (chroma_persist_dir, bm25_persist_path, document manifest).
    - Create a test text file with known content (~1000 chars).

    - Test duplicate ingestion: ingest same file twice → should return existing, not create duplicates.
    - Test delete: ingest file → delete document → verify counts are 0 in all stores.
    - Test verify_sync: ingest file → call verify_sync → confirm in_sync is True.
    - Override Settings in tests to use tmp_path for all storage paths.
  </action>
