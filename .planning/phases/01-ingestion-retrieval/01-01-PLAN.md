  <action>
    Create a BaseExtractor Protocol and 4 concrete extractors. Each extractor takes a file path and returns extracted text + metadata.

    **extractors/base.py:**
    - Define `ExtractedDocument(BaseModel)`: text (str), metadata (dict) containing at minimum source_file, file_type, and optionally page_count.
    - Define `BaseExtractor(Protocol)`: method `extract(self, file_path: Path) -> ExtractedDocument`. Also define `supported_extensions: list[str]` class attribute.
    - Define `ExtractorRegistry` class: maps file extensions to extractor instances. Method `get_extractor(file_path: Path) -> BaseExtractor` looks up by extension, raises `ExtractionError` if unsupported. Method `extract(file_path: Path) -> ExtractedDocument` is convenience — gets extractor and calls extract.

    **extractors/pdf.py — PDFExtractor:**
    - Uses PyMuPDF (`fitz`) to extract text page-by-page.
    - `supported_extensions = [".pdf"]`
    - Extract text from each page using `page.get_text("text")`.
    - Join pages with `\n\n` separator to preserve page boundaries.
    - Store `page_count` in metadata.
    - Handle empty PDFs (0 pages or no text) — raise ExtractionError with descriptive message.
    - Handle corrupted PDFs — catch fitz exceptions, wrap in ExtractionError.

    **extractors/markdown.py — MarkdownExtractor:**
    - `supported_extensions = [".md", ".markdown"]`
    - Read file as UTF-8 text.
    - Return raw markdown text (do NOT convert to HTML — the chunker will handle markdown structure). Markdown is already readable text; stripping formatting would lose section headers that inform chunking.
    - Store `file_size` in metadata.

    **extractors/html.py — HTMLExtractor:**
    - Uses BeautifulSoup4 with `html.parser`.
    - `supported_extensions = [".html", ".htm"]`
    - Strip all tags but preserve text content. Use `soup.get_text(separator="\n", strip=True)`.
    - Remove `<script>`, `<style>`, `<nav>`, `<footer>`, `<header>` elements BEFORE extracting text (these are noise).
    - Store `title` from `<title>` tag in metadata if present.

    **extractors/plaintext.py — PlaintextExtractor:**
    - `supported_extensions = [".txt", ".text"]`
    - Read file as UTF-8 with fallback to latin-1 if UTF-8 fails.
    - Return text as-is.
    - Store `file_size` in metadata.

    **extractors/__init__.py:**
    - Create a `create_default_registry() -> ExtractorRegistry` function that registers all 4 extractors.

    **tests/unit/test_extractors.py:**
    - Create small test fixtures inline (not external files): a tiny PDF (use fitz to create programmatically), a markdown string, an HTML string, a plain text string.
    - Write to tmp_path in each test.
    - Test each extractor: verify text is extracted, metadata is populated, and expected content is present.
    - Test ExtractorRegistry: verify correct extractor selected by extension, verify ExtractionError on unsupported extension (.docx).
    - Test error handling: empty file, corrupted file (random bytes as PDF).
  </action>
