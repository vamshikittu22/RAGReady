  <action>
    1. Create `src/types/api.ts` — TypeScript interfaces matching the backend API models:
       ```typescript
       // Match exactly what the FastAPI backend returns
       export interface Citation {
         chunk_text: string;
         document_name: string;
         page_number: number | null;
         relevance_score: number;
       }

       export interface QueryResponse {
         answer: string;
         citations: Citation[];
         confidence: number;
         refused: false;
       }

       export interface RefusalResponse {
         refused: true;
         reason: string;
         confidence: number;
         answer: "";
         citations: [];
       }

       export type QueryResult = QueryResponse | RefusalResponse;

       export interface UploadResponse {
         document_id: string;
         filename: string;
         file_type: string;
         chunk_count: number;
       }

       export interface DocumentInfo {
         document_id: string;
         filename: string;
         file_type: string;
         chunk_count: number;
         created_at: string;
       }

       export interface DocumentListResponse {
         documents: DocumentInfo[];
         count: number;
       }

       export interface DeleteResponse {
         deleted: string;
       }

       export interface HealthResponse {
         status: string;
         version: string;
         llm_model: string;
         fallback_model: string | null;
         document_count: number;
         phoenix_enabled: boolean;
       }

       // Chat message type for UI state
       export interface ChatMessage {
         role: 'user' | 'assistant';
         content: string;
         citations?: Citation[];
         confidence?: number;
         refused?: boolean;
         refusalReason?: string;
       }
       ```

    2. Create `src/lib/api.ts` — API client module:
       - Use `const API_BASE = import.meta.env.VITE_API_URL || 'http://localhost:8000'`
       - Create `ApiError` class extending Error with `status` property
       - Create a generic `fetchApi<T>(path, options)` function that handles errors (throws ApiError with status + detail)
       - Export `api` object with methods: `query(question)`, `getDocuments()`, `uploadDocument(file)`, `deleteDocument(id)`, `getHealth()`
       - CRITICAL for uploadDocument: Do NOT set Content-Type header — let browser set multipart boundary. Use FormData, not JSON.
       - Define `queryKeys` constant for TanStack Query key consistency: `{ documents: ['documents'] as const, health: ['health'] as const }`

    3. Create `src/hooks/use-query-rag.ts`:
       - Uses `useMutation` from TanStack Query with `api.query(question)` as mutationFn
       - Returns `{ ask, data, isPending, isError, error, reset }`
       - `ask` is the `mutate` function renamed for clarity

    4. Create `src/hooks/use-documents.ts`:
       - `useQuery` for fetching document list with queryKey `queryKeys.documents`
       - `useMutation` for upload with `onSuccess` invalidating `queryKeys.documents`
       - `useMutation` for delete with `onSuccess` invalidating `queryKeys.documents`
       - Returns `{ documents, count, isLoading, upload, deleteDoc, isUploading, isDeleting }`

    5. Create `src/hooks/use-health.ts`:
       - `useQuery` with queryKey `queryKeys.health`, refetchInterval: 30000 (poll every 30s)
       - Returns `{ health, isHealthy, isLoading }`
  </action>
