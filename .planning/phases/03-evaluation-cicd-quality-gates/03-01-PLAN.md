    Fixtures (all session-scoped for performance):

    - `eval_settings(tmp_path_factory)` — Settings with tmp_path storage dirs (use `tmp_path_factory.mktemp("eval")` since session scope requires `tmp_path_factory` not `tmp_path`)
    - `eval_pipeline(eval_settings)` — IngestionPipeline with all 3 test fixture documents ingested
    - `eval_hybrid_retriever(eval_pipeline)` — HybridRetriever wired to ingested stores
    - `eval_dense_retriever(eval_pipeline)` — DenseRetriever only (for naive baseline)
    - `golden_dataset()` — Load and return parsed golden_dataset.json
    - `eval_retrieval_results(eval_hybrid_retriever, golden_dataset)` — Run each non-refusal golden dataset question through the hybrid retriever. Returns list of (question, list[ScoredChunk]) tuples. This does NOT run the full RAG chain (no LLM needed) — retrieval-only.
    - `eval_dense_results(eval_dense_retriever, golden_dataset)` — Same but through DenseRetriever for naive baseline comparison.
    - `eval_rag_responses(eval_pipeline, eval_hybrid_retriever, golden_dataset)` — Generate synthetic QueryResponse/RefusalResponse objects from retrieval results WITHOUT calling any LLM. Logic:
      - For each question, run hybrid retrieval
      - If `should_refuse=True` in golden entry OR retrieval returns no chunks: create RefusalResponse
      - Otherwise: create QueryResponse with answer text = concatenation of top-3 chunk texts (synthetic), citations built from ScoredChunk metadata (document_name, chunk_text, page, score)
      - This ensures deterministic, CI-safe evaluation. Real LLM responses are only tested locally via separate `@pytest.mark.ollama` marked tests.

    IMPORTANT: Do NOT try to connect to Ollama in conftest. All fixtures must be deterministic and work offline. The synthetic response generation strategy ensures faithfulness/citation metrics test the measurement pipeline itself (not the LLM quality — that's a local-only test).

    - `embedding_model()` — Load `all-MiniLM-L6-v2` SentenceTransformer once for all metric computations.

    Fixture scope: Use `session` scope for all heavy fixtures (pipeline, retrieval results, embedding model).
