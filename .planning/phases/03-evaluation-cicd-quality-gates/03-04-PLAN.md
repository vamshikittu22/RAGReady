  <action>
    Create `.github/workflows/ci.yml` with the following job structure:

    **Trigger:** `on: [push, pull_request]` targeting `main` branch.

    **Job 1: lint-and-typecheck**
    - Runs on: `ubuntu-latest`
    - Python: `3.12` (not 3.14 — GitHub Actions may not have 3.14 yet; 3.12 is the minimum supported version per pyproject.toml)
    - Steps:
      1. Checkout code
      2. Set up Python 3.12
      3. Install dependencies: `pip install -e ".[dev]"`
      4. Run ruff: `ruff check src/ tests/`
      5. Run mypy: `mypy src/ --ignore-missing-imports` (use --ignore-missing-imports since some deps like chromadb lack type stubs)

    **Job 2: unit-tests**
    - Runs on: `ubuntu-latest`
    - Python: `3.12`
    - Steps:
      1. Checkout code
      2. Set up Python 3.12
      3. Install dependencies: `pip install -e ".[dev]"`
      4. Run unit tests: `pytest tests/unit/ -v --tb=short`

    **Job 3: integration-tests**
    - Runs on: `ubuntu-latest`
    - Python: `3.12`
    - Needs: `unit-tests` (run after unit tests pass)
    - Steps:
      1. Checkout code
      2. Set up Python 3.12
      3. Install dependencies: `pip install -e ".[dev]"`
      4. Cache HuggingFace models: Use `actions/cache@v4` with path `~/.cache/huggingface` and key based on `hashFiles('pyproject.toml')`
      5. Run integration tests: `pytest tests/integration/ -v --tb=short`

    **Job 4: evaluation**
    - Runs on: `ubuntu-latest`
    - Python: `3.12`
    - Needs: `integration-tests` (run after integration tests pass)
    - Steps:
      1. Checkout code
      2. Set up Python 3.12
      3. Install dependencies: `pip install -e ".[dev,eval]"`
      4. Cache HuggingFace models: Use `actions/cache@v4` with path `~/.cache/huggingface` and key `huggingface-${{ hashFiles('pyproject.toml') }}`
      5. Create reports directory: `mkdir -p reports`
      6. Run evaluation tests: `pytest tests/evaluation/ -v --tb=short -m "evaluation and not slow and not ollama"` — this runs only the 4 CI-safe quality gate tests + 3 benchmark tests (skips LLM-dependent tests)
      7. Upload evaluation report artifact:
         ```yaml
         - uses: actions/upload-artifact@v4
           if: always()
           with:
             name: evaluation-report
             path: reports/
             retention-days: 30
         ```

    **Important CI notes:**
    - Each job should use `pip install -e ".[dev]"` (editable install with dev deps)
    - The evaluation job also needs `.[eval]` for ragas dependency
    - Tests marked `@pytest.mark.ollama` are EXCLUDED from CI (no LLM available)
    - The 4 CI-safe quality gate tests (context recall, context precision, refusal accuracy, citation accuracy) DO run and WILL block the PR if thresholds fail
    - The `if: always()` on artifact upload ensures reports are saved even if tests fail

    **Note on PR blocking:** The CI workflow causes the evaluation job to FAIL on threshold violations. To actually block PR merging, the user must configure GitHub branch protection rules (Settings → Branches → main → Require status checks) to require the `evaluation` job. Document this in a comment at the top of ci.yml:
    ```yaml
    # NOTE: To enforce PR blocking, enable branch protection in GitHub repo settings:
    #   Settings → Branches → main → Require status checks → evaluation
    ```

    **Update .gitignore:**
    Add `reports/` directory if not already present.
