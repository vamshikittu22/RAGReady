  <action>
    **Step 1: test_quality_gates.py** — pytest test file asserting all metric thresholds.

    Mark the entire module with `@pytest.mark.evaluation` so CI can target it specifically.

    **CI-safe tests (retrieval-only, run on every PR):**
    - `test_context_recall_above_threshold` — assert context recall >= 0.75
    - `test_context_precision_above_threshold` — assert context precision >= 0.70
    - `test_refusal_accuracy_above_threshold` — assert refusal accuracy >= 0.90
    - `test_citation_accuracy_above_threshold` — assert citation accuracy >= 0.95

    These 4 tests use `eval_retrieval_results`, `eval_rag_responses`, `golden_dataset`, and `embedding_model` fixtures from conftest. They work with synthetic responses (no LLM needed).

    **Local-only tests (require real LLM responses, marked `@pytest.mark.ollama`):**
    - `test_faithfulness_above_threshold` — assert faithfulness >= 0.85
    - `test_answer_relevancy_above_threshold` — assert answer relevancy >= 0.80
    - `test_hallucination_rate_below_threshold` — assert hallucination < 0.15 (= 1 - 0.85 faithfulness)

    These 3 tests are marked `@pytest.mark.ollama` and `@pytest.mark.slow`. They ONLY produce meaningful results with real LLM-generated responses. In CI (which excludes `@pytest.mark.ollama`), they are skipped.

    Each test should print the actual score in the assertion message:
    ```python
    assert score >= 0.75, f"Context Recall {score:.3f} below threshold 0.75"
    ```

    Use a shared fixture `eval_metrics` (session-scoped, defined in this file or conftest) that computes all applicable metrics once and each test reads from the dict. The fixture should:
    1. Use fixtures from conftest (golden_dataset, eval_retrieval_results, eval_rag_responses, embedding_model)
    2. Compute each metric using the corresponding compute_* function
    3. Return a dict: `{"context_recall": 0.82, "context_precision": 0.75, ...}`
    4. Write results to `reports/evaluation-{date}.json` for artifact collection (create `reports/` if needed)

    **Step 2: Update pyproject.toml**

    Add `eval` optional dependency group:
    ```toml
    [project.optional-dependencies]
    eval = [
        "ragas>=0.4.0",
    ]
    ```

    Add pytest markers to existing `[tool.pytest.ini_options]`:
    ```toml
    markers = [
        "evaluation: marks tests as evaluation pipeline tests",
        "slow: marks tests as slow (LLM-dependent)",
        "ollama: marks tests requiring Ollama server",
    ]
    ```

    Add `reports/` to `.gitignore` if not already present.
  </action>

  <action>
    **Step 1: scripts/benchmark.py** — The single source of truth for naive vs hybrid benchmark comparison.

    A standalone CLI script AND importable module:

    ```python
    """Benchmark: Naive (dense-only) vs Hybrid retrieval comparison.

    Runs the golden dataset through both DenseRetriever and HybridRetriever,
    computes metrics for each, and generates a comparison report.

    Usage: python scripts/benchmark.py [--output reports/]
    """
    ```

    Export a `run_benchmark(...)` function that:
    1. Takes pipeline, golden_dataset, and embedding_model as parameters
    2. Creates both retrievers (DenseRetriever and HybridRetriever) from pipeline
    3. Runs all non-refusal questions through both retrievers
    4. Computes Context Recall and Context Precision for both using metric functions
    5. Returns a dict with results:
       ```python
       {
         "timestamp": "...",
         "metrics": {
           "naive": {"context_recall": 0.65, "context_precision": 0.55},
           "hybrid": {"context_recall": 0.82, "context_precision": 0.75}
         },
         "improvement": {"context_recall": "+26%", "context_precision": "+36%"},
         "winner": "hybrid"
       }
       ```

    Also export `print_benchmark_table(results)` for formatted stdout output:
    ```
    +---------------------+--------+--------+
    | Metric              | Naive  | Hybrid |
    +---------------------+--------+--------+
    | Context Recall      | 0.65   | 0.82   |
    | Context Precision   | 0.55   | 0.75   |
    +---------------------+--------+--------+
    ```

    The `if __name__ == "__main__":` block should:
    1. Parse CLI args (--output directory, defaults to `reports/`)
    2. Create IngestionPipeline with temp directory, ingest fixture documents
    3. Load golden dataset, load embedding model
    4. Call `run_benchmark()`
    5. Call `print_benchmark_table()`
    6. Save JSON report to output directory
    7. Clean up temp directory

    Use argparse, tempfile.mkdtemp(), import from tests.evaluation.metrics.

    **Step 2: tests/evaluation/test_benchmark.py** — Pytest tests that import from scripts/benchmark.py (single source of truth).

    Mark with `@pytest.mark.evaluation`.

    Tests:
    - `test_hybrid_beats_naive_on_context_recall(benchmark_results)` — hybrid context recall > naive context recall
    - `test_hybrid_beats_naive_on_context_precision(benchmark_results)` — hybrid precision > naive precision
    - `test_benchmark_report_generated(benchmark_results)` — benchmark results dict has all required keys

    Use a `benchmark_results` fixture (session-scoped) that:
    1. Imports `run_benchmark` from `scripts.benchmark`
    2. Uses `eval_pipeline`, `golden_dataset`, and `embedding_model` fixtures from conftest
    3. Calls `run_benchmark(pipeline, golden_dataset, embedding_model)`
    4. Returns the results dict

    This ensures NO duplication — test_benchmark.py delegates all logic to scripts/benchmark.py.

    **Step 3: scripts/evaluate.py** — CLI evaluation runner.

    A standalone script that:
    1. Ingests test fixture documents into a temp directory pipeline
    2. Runs all golden dataset questions through the pipeline (retrieval only)
    3. Generates synthetic responses (same logic as conftest)
    4. Computes all applicable metrics (context recall, precision, refusal accuracy, citation accuracy)
    5. If Ollama is available, also computes faithfulness, answer relevancy, hallucination rate
    6. Prints results table to stdout
    7. Saves JSON report to `reports/`

    Usage: `python scripts/evaluate.py [--output reports/] [--with-ollama]`

    Use argparse. Import from tests.evaluation.metrics modules.
  </action>
